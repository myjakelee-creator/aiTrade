개발 단계 이해

단계상태설명
1. 화면 만들기거의 완료StockBoard HTML, 셀폭, 캔들, 후보5, TOP100 표시
2. REST 장마감 데이터거의 완료거래대금, OHLC, VWAP, 시장수급 일부 연결
3. 파일 구조 정리지금 단계통합 Python 1개를 Python 4개 구조로 정리
4. OpenAPI 실시간다음 핵심외선, 체결, 호가, 잔량, 큰손/KRT 연결
5. 시그널 두뇌이후 핵심Signal → Ranking → Strategy로 후보5 선정

Python 4개로 줄였을 때의 장점

파일이 너무 많으면 헷갈리고, 하나뿐이면 너무 복잡해집니다. 그래서 지금은 데이터 수집 / 저장 / 계산 / 서버 4개로만 나누는 것이 가장 현실적입니다.
장점설명
이해가 쉽다각 파일의 역할이 명확하다.
수정 범위가 작다REST를 고칠 때 HTML이나 Engine을 건드리지 않아도 된다.
실시간 확장이 쉽다OpenAPI+ COM 데이터를 data_provider와 store에 붙이면 된다.
후보5 로직이 분리된다시그널 연구실 이론을 stockboard_engine.py에 집중시킬 수 있다.
